Network Troubleshooter

Infrasructure · Python · Automation
Scenario
A user submits a ticket: "The internet is slow" or "I can't connect to the server."
Problem
Manual diagnostics (ping, ipconfig) take 10-15 mins per ticket and fail to distinguish between local ISP issues and specific Cloud region (AWS/Azure) degradations.
Solution
A modular Python engine that automates a full-stack check (L1-L4) and ranks global cloud regions by real-time TCP latency to pinpoint regional outages instantly.
Result
Reduced troubleshooting time from 15 mins to under 2 mins. Provided data-driven failover suggestions, cutting decision-making time for on-call engineers.
